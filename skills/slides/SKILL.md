[현재 과업: 차시 강의 슬라이드 개요(PPT 아웃라인) 작성]
제공된 강의계획서의 해당 주차 목표를 상속하여 슬라이드 개요를 작성한다. 새 목표를 만들지 않는다.

[분량 — 반드시 지킬 것]
- 슬라이드 수는 차시 수업 시간에 비례한다: **1시간당 약 20장**(예: 2시간 → 약 40장).
- 사용자 메시지에 제시된 목표 장수·슬라이드 제목 목록을 **그대로 모두** 채운다. 목표보다 적게 만들지 않는다.
- 한 개념을 정의·예시·비교·적용으로 여러 슬라이드에 나눠 밀도를 낮춘다.

[오리엔테이션 — 기본 최소화]
- 도입·오리엔테이션(과목 소개·평가 안내·자기소개 등)은 **기본적으로 최소(1~2장)** 로 하고, 나머지는 곧바로 본론(이론·개념·사례·적용)에 배분한다.
- 예외: 사용자(교수자) 요청사항에 오리엔테이션을 길게/자세히 하라는 지시가 있을 때만 그만큼 늘린다.

[슬라이드 원칙]
- 한 슬라이드에 하나의 메시지, 텍스트 최소화(불릿 3~5개).
- 발표자 노트·화자 스크립트는 넣지 않는다.

[레이아웃 — 회사 양식에 매핑]
- 아래 3가지만 사용한다(그 외 스타일은 렌더되지 않음):
  · **섹션 표지** — 도입·구간 전환·결론(큰 제목)
  · **2단** — 두 항목을 좌·우로 비교(본문 불릿이 좌/우 2칸에 자동 분배)
  · **콘텐츠** — 일반 제목 + 불릿
- 색·폰트·로고는 회사 PPT 양식이 결정하므로 색상 코드를 지정하지 않는다.

출력 구조:
1. 맨 앞에 **차시 학습목표**(ABCD·인지수준 태그·지지 강좌목표 G#)와 **슬라이드 구성 개요** 표(구간 | 슬라이드 수 | 시간).
2. 슬라이드별 개요 — 각 슬라이드를 다음 형식으로:

### 슬라이드 N — [제목]
- **레이아웃 제안**: (섹션 표지 / 2단 / 콘텐츠 중 택1)
- **핵심 메시지(1개)**: …
- **본문 개요**: 불릿 3~5개

3. 흐름: 목표 제시 → Hook → 선수지식 연결 → 개념(토대→응용, 여러 장 분절) → 형성평가(1~2장) → 요약·다음 차시 예고.

## 첨삭 반영 규칙 (누적, 날짜순)
- (2026-07-09) 1시간당 20장(2시간=40장)으로 상향. 발표자 노트 제거. 개수는 2단계(제목 목록 → 상세) 생성으로 보장.
- (2026-07-09) 오리엔테이션은 기본 최소(1~2장), 본론(이론·개념) 중심. 길게 필요할 때만 교수자 요청사항으로 지시.
- (2026-07-14) 목표 장수(예: 2시간=40장)를 반드시 채운다. 개요 뒤 STEP 4-2(노트북LM 렌더 코드)·4-3(비주얼 원고)에서 이 개요를 그대로 슬라이드/시각물로 변환하므로, 각 슬라이드의 '핵심 메시지 1개'와 '본문 개요 불릿'을 빠짐없이 적는다.
